Promo Bot

A Python automation system that monitors Mercado Livre and Shopee for deals, filters them by discount, price range, category and coupon availability, generates branded product images and delivers curated affiliate promotions to multiple WhatsApp groups simultaneously — fully hands-off after setup.

  • Type: Affiliate Marketing Automation
  • Stack: Python · WhatsApp API · Shopee Affiliate API · Telethon
  • Year: 2024
  • Category: Automation · Affiliate
GitHub
01
Problem

Running WhatsApp promo groups manually meant hours of searching deals across Mercado Livre and Shopee, converting links to affiliate URLs and formatting messages by hand.

02
Solution

Python system that polls ML + Shopee APIs, filters by discount %, price range, national/imported flag and coupon ratio, generates branded images and pushes affiliate-linked posts to multiple WhatsApp groups.

03
Impact

Zero manual effort after setup — multiple groups, each with its own category config, receive curated deals 24/7 with deduplication and ML coupon mirroring from Telegram channels.

Project Description

- Polls Mercado Livre and Shopee APIs on a configurable interval, filtering by minimum discount, price range and national/imported flag;
- Multi-group routing: each WhatsApp group has its own category JSON with keywords, so tech deals go to one group and beauty deals to another;
- Coupon-ratio engine (e.g. every 10 posts, at least 1 must have a coupon) with automatic fallback;
- Telegram userbot mirrors coupon posts from promo channels and re-delivers them to WhatsApp with affiliate links;
- Image generator wraps product photos in a branded frame (per-group configurable);
- Deduplication cache with 120h TTL prevents the same deal from being sent twice;
- Tier distribution system prioritises high-value deals (60% high / 30% mid / 10% low) per cycle.

Multi-group Routing

Each WhatsApp group has its own category config — tech deals, beauty, electronics — all managed from a single bot instance.

Telegram Mirror

Userbot watches curated Telegram promo channels and re-delivers ML coupon posts to WhatsApp with affiliate links injected automatically.

Smart Filtering

Discount threshold, price range, coupon ratio, tier distribution and a 120h dedup cache ensure only quality deals reach the groups.

Built in Godot 4 with statically-typed GDScript — includes 4 difficulty modes, an in-game educational survey, a real-time global leaderboard, and standalone builds for Windows and Android.

A social awareness campaign needed high engagement and public opinion data — static materials failed to retain attention or generate actionable insights.

Godot 4 advergame with a 3-act narrative, 4 difficulty modes, a 6-question in-game survey, and a global online leaderboard — all in ~600 KB with zero external dependencies.

BE HUMBLE Bot

A Python Discord bot built to automate the full member onboarding flow on a Discord server. When a new member joins, the bot assigns a restricted role, sends a DM welcome message, collects four key pieces of information (nickname, full name, birthday and availability), reports them to a moderators-only channel, saves data to an XLSX spreadsheet, then renames the user and grants them full server access — all without admin intervention.

  • Type: Discord Bot · Community Management
  • Stack: Python · discord.py · openpyxl
  • Category: Automation · Bot Development
  • Year: 2024
  • Commands: /Ping · /Limpar · /Embed · /Registro
GitHub
01
Problem

Onboarding new Discord members manually was slow and error-prone — admins had to collect info, rename users, assign roles and log data one by one for every new arrival.

02
Solution

Python bot that automates the entire pipeline: restricted role on join → DM survey → moderator report → XLSX data logging → user rename and full role grant, with daily birthday checks as a bonus.

03
Impact

Zero admin effort per new member — the full onboarding flow runs autonomously; daily birthday messages boost community engagement automatically.

Project Description

- On member join: assigns a restricted role limiting access to an instructions channel, then sends a DM welcome message;
- DM survey collects 4 fields: Nickname, Full Name, Birthday and Available Hours;
- After survey completion, sends a formatted embed to a moderators-only channel with all collected data;
- Saves nickname and birthday to an XLSX spreadsheet (openpyxl) for persistent storage;
- Renames the user with the provided nickname, removes the restricted role and grants the full-access role — all automatically;
- Daily birthday checker reads the spreadsheet and sends an embed mention in the general channel for any member celebrating that day;
- 4 slash commands: /Ping (latency), /Limpar (bulk delete, mods only), /Embed (custom embed builder), /Registro (manual survey trigger without rename/role change).

Automated Onboarding

Full pipeline from join event to full server access — restricted role, DM survey, moderator notification, XLSX logging, rename and role upgrade, all without a single admin click.

Birthday System

Daily async task reads the XLSX sheet and fires an embed mention in the general channel for every member whose birthday matches the current date.

Slash Commands

Four utility commands: latency check, moderator-only bulk message deletion, custom embed builder and a manual registration flow for existing members.
